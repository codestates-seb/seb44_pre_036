# 6. git commit rule
| 커밋 유형 | 의미 |
|:--------:|:-----------------------------------------:|
|   🎉    | project 시작 |
|   ✨    | 새로운 기능 추가 |
|   🐛    | 버그 수정 |
|   📝    | README 등 문서 수정 |
|   🎨    | 코드 구조 관련 변경사항(순서 등등) |
|   ♻️    | 코드 리팩토링 |
|   ➕    | package.json에 dependency 추가 |
|   🙈    | .gitignore에 파일 추가 |
|   💄    | 컴포넌트 생성, CSS 등 사용자 UI 디자인 변경 |
|   💡    | 필요한 주석 추가 및 변경 |
|   🚚    | 파일 또는 폴더 명을 수정하거나 옮기는 작업만인 경우 |
|   🔥    | 파일 or 폴더 or 코드를 삭제하는 작업만 수행한 경우 |
|  👽️   | API 변경의 경우 |
|   🚑    | hotfix |
|   🗃️    | DB 관련 변경 사항 |
|   ⚡️    | 성능 개선 |
|   🚨    | 에러 고쳤을 때 |
|   🚀    | 배포 |
|   🔧    | config 파일 생성 또는 수정 |
|   🔐    | .env 파일 생성 또는 수정 |
